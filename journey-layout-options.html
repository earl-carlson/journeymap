Journey Layout Options

All four approaches using Hub journeys as the test case (6 journeys). Evaluate at 1200×675px — actual slide scale.

Option A — Multi-slide table split
Keep the table format. Split any area with more than 4 journeys across multiple slides. Simple, no redesign, fast to apply everywhere. The table stays readable but the format doesn't do much for the content.

Option C — Swimlane / stage map
Each journey is a horizontal row. Columns map to journey stages. Shows the flow and lets you compare stages across journeys at a glance. Dense at 6 rows but readable. Best for areas where stages align (like Admin). Tightest at scale.

Option D — One journey per slide, full detail
Each journey gets its own slide. Stage columns show label + description. The most useful for stakeholders who want to actually understand what a journey involves. Adds ~20 slides but each one is self-contained and presentation-ready. Showing two examples.
Hub · Journey
Image discovery
Developer
01
Search Hub
Developer searches for a base image or tool using keywords, categories, or use-case filters.
02
Filter by trusted content
Filters by Docker Official Images, Verified Publisher, or Sponsored OSS to narrow to trustworthy results.
03
Browse and evaluate
Opens a repository, reads the overview, checks tags, reviews vulnerability and health status.
04
Pull image
Copies the pull command and runs it locally. Image is now available in Docker Desktop or via CLI.
Hub · Journey
Hub Publisher
Developer Platform Engineer
01
Build locally
Builds the image locally with Docker CLI. Tags it with the correct namespace and version.
02
Push to Hub
Authenticates and pushes the image to the Docker Hub repository. Sets visibility — public or private.
03
Write repo overview
Adds a README so others know how to use the image. No enforced structure — consistency varies.
04
Set up automation
Configures webhooks for downstream CI triggers. Sets repository access for the org namespace.
05
Monitor usage
Checks pull counts, storage usage, and tag activity via Hub insights over time.
